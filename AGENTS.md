# Project instructions for Codex

## Git

- Never run `git commit` or `git push` without the user's explicit go-ahead, every time. Do not treat a prior approval as standing permission for later commits in the same session.
- When executing a multi-task implementation plan (e.g. via subagent-driven-development), skip the per-task "commit" steps. Implement and test all tasks with changes left uncommitted, then ask the user once at the end whether/how to commit.
